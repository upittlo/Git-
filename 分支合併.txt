### 預設master就是分支

1. 建立新分支
git branch develop
(develop為分支名稱)

2. 查看所有分支
git branch

3. 切換分支

git checkout "branch name"

4. 建立分支且直接切換

git checkout -b 'branch name'


5. 刪除分支 (不可刪除所在分支)

git branch -d 'name'

(若分支有新的commit且還沒被合併, -d 無法刪除)

git branch -D 'name'

無是是否有新commit,強制刪除

6. 誤刪分支復原

刪除時會有id 

git branch 'name' 'id' 即可

7. 分支合併

先接換到"主要"分支上 (切換前要先commit)

git checkout "main branch"

git merge "預合併分支"

預合併分支會不有任何改變，主分支將被修改

8. 回到合併前狀態

git reset --hard ORIG_HEAD

9. 分支合併後建議處理方式

建議只使用單方向合併 (往MASTER合併)

可以砍掉合併後的branch
並且在最新版重新開啟分支
用 git branch -d 'branch name' 刪除已合併分支


10. 查詢版本線圖

git log --oneline --graph

or 

git log --graph

11. 衝突

改到同一檔案同一行才會有衝突

(a). 也可以放棄合併 git merge --abort
(用於不確定其他人修改邏輯的狀況)


(b). 可用vi 進入衝突檔案看細節
並且修改 
須把 <<<   >>> HEAD等等都修掉
修改完後ESC  :wq

git add "檔名"
git commit  即可完成合併


12. fast-forward

快轉機制

若要合併的分支為現有分支的未來版本 (分支有改過，master沒改過)
快轉機制會直接把branch變成主支 (直接跳一版)
造成版本線圖不易辨識

(用於本地分支與遠端合併)

13. 避免fast forward的合併方法

git merge "branch name" --no-ff
則會用一般方式合併

(用於意義上兩條不同分支)
